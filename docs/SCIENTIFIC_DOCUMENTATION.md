# SARdine Scientific Documentation
## Mathematical Algorithms and References

### Overview
This document provides comprehensive scientific documentation for all SAR processing algorithms implemented in SARdine, including mathematical equations, physical principles, and scientific references.

## 1. SAR Fundamentals

### 1.1 Radar Equation
The fundamental radar equation for SAR backscatter is:

```
σ⁰ = (4π)³ × R⁴ × DN² / (λ² × Pt × G² × τ × c × sin(θ))
```

Where:
- σ⁰: Normalized radar cross-section (m²/m²)
- R: Slant range (m)
- DN: Digital Number (raw pixel value)
- λ: Radar wavelength (m)
- Pt: Transmitted power (W)
- G: Antenna gain
- τ: Pulse duration (s)
- c: Speed of light (m/s)
- θ: Incidence angle (rad)

**Reference**: Cumming & Wong, "Digital Processing of Synthetic Aperture Radar Data", 2005

### 1.2 Sentinel-1 C-Band Parameters
- **Center Frequency**: 5.405 GHz (exact)
- **Wavelength**: λ = c/f = 299,792,458 / 5.405e9 = 0.0555 m
- **Bandwidth**: 56.5 MHz
- **Orbital Height**: ~693 km (nominal)
- **Orbital Velocity**: ~7.5 km/s (varies with position)

**Reference**: ESA Sentinel-1 Product Specification, EOP-SM/1087/PS

## 2. TOPSAR Processing

### 2.1 Azimuth Deramp Phase Correction
For TOPSAR burst processing, the azimuth phase correction is:

```
φ(t_az, t_r) = 2π × f_dc × t_az + π × k_a × t_az² + φ_steering(t_az)
```

Where:
- t_az: Azimuth time relative to burst center (s)
- t_r: Range time (s)
- f_dc: Doppler centroid frequency (Hz)
- k_a: Azimuth FM rate (Hz/s)
- φ_steering: Steering phase for beam steering (rad)

**Implementation**: `src/core/deburst.rs:calculate_deramp_phase()`
**Reference**: ESA-EOPG-CSCOP-TN-0009 "Sentinel-1 IPF Algorithms"

### 2.2 Azimuth Time Calculation
**CRITICAL FIX**: Previously used hardcoded 7 km/s velocity approximation.
Now uses precise orbit velocity:

```
t_az = (line - line_center) × Δ_az / v_sat
```

Where:
- line: Current azimuth line
- line_center: Burst center line
- Δ_az: Azimuth pixel spacing (m)
- v_sat: Actual satellite velocity from orbit state vectors (m/s)

**Reference**: ESA Sentinel-1 Level 1 Detailed Algorithm Definition

### 2.3 Burst Overlap Weighting
Cosine-squared weighting for seamless burst merging:

```
w(i) = {
  [sin²(π × i / N_overlap)]       for i < N_overlap
  1                               for N_overlap ≤ i ≤ N_burst - N_overlap
  [sin²(π × (N_burst - i) / N_overlap)]  for i > N_burst - N_overlap
}
```

**Reference**: IEEE TGRS papers on TOPSAR processing

## 3. Radiometric Calibration

### 3.1 Calibration Equation
Conversion from Digital Numbers to backscatter:

```
σ⁰(dB) = 10 × log₁₀(DN²) + 10 × log₁₀(calibration_LUT)
```

For linear scale:
```
σ⁰ = DN² × calibration_LUT
```

**Implementation**: `src/core/calibrate.rs`
**Reference**: ESA Sentinel-1 Product Specification, Section 2.2.1

### 3.2 Noise Equivalent Sigma Zero (NESZ)
Quality assessment using noise floor:

```
NESZ(dB) = 10 × log₁₀(noise_LUT)
```

**Reference**: ESA Sentinel-1 Thermal Denoising of Products Generated by IPF

## 4. Terrain Correction

### 4.1 Range-Doppler Terrain Correction
The fundamental equation for terrain correction:

```
(R, f_d) = f(lat, lon, h, t)
```

Where:
- R: Slant range (m)
- f_d: Doppler frequency (Hz)
- lat, lon: Geographic coordinates
- h: Elevation (m)
- t: Azimuth time (s)

**Implementation**: `src/core/terrain_correction.rs`
**Reference**: Small & Schubert, "Guide to SAR Geocoding", Sandia Report 2019

### 4.2 Iterative Solution
Newton-Raphson iteration for pixel positioning:

```
[Δlat]   = -J⁻¹ × [R_calc - R_meas]
[Δlon]            [f_d_calc - f_d_meas]
```

Where J is the Jacobian matrix:
```
J = [∂R/∂lat   ∂R/∂lon  ]
    [∂f_d/∂lat ∂f_d/∂lon]
```

**Reference**: Curlander & McDonough, "Synthetic Aperture Radar: Systems and Signal Processing"

### 4.3 DEM Interpolation
Bilinear interpolation for elevation lookup:

```
h(lat, lon) = h₁₁(1-t)(1-u) + h₁₂(1-t)u + h₂₁t(1-u) + h₂₂tu
```

Where t, u are fractional coordinates within DEM cell.

**Implementation**: `src/io/dem.rs:resample_dem()`

## 5. Multilooking

### 5.1 Intensity Multilooking
For N_rg × N_az multilooking:

```
I_ML = (1/(N_rg × N_az)) × Σᵢ₌₁ᴺʳᵍ Σⱼ₌₁ᴺᵃᶻ |s(i,j)|²
```

**Implementation**: `src/core/multilook.rs`
**Reference**: Lee & Pottier, "Polarimetric Radar Imaging", 2009

### 5.2 Equivalent Number of Looks (ENL)
Statistical measure of multilooking effectiveness:

```
ENL = μ² / σ²
```

Where μ is mean intensity and σ² is variance.

## 6. Speckle Filtering

### 6.1 Enhanced Lee Filter
Adaptive speckle filter preserving edges:

```
I_filtered = μ + k × (I - μ)
```

Where:
```
k = (1 - Cᵤ²/Cᵢ²) / (1 + Cᵤ²)
```

- Cᵤ: Coefficient of variation for uniform areas
- Cᵢ: Coefficient of variation for local area

**Implementation**: `src/core/speckle_filter.rs`
**Reference**: Lopes et al., IEEE TGRS, 1993

## 7. Geocoding

### 7.1 Map Projection
World Geodetic System 1984 (WGS84) to UTM projection:

```
E = k₀ × N × [A + (1-T+C) × A³/6 + (5-18T+T²+72C-58e'²) × A⁵/120]
N = k₀ × [M + N × tan(φ) × (A²/2 + (5-T+9C+4C²) × A⁴/24)]
```

**Reference**: Snyder, "Map Projections: A Working Manual", USGS Professional Paper 1395

## 8. Quality Assessment

### 8.1 Backscatter Validation Ranges
Physically reasonable ranges for different targets:

- **Water**: σ⁰ < -15 dB
- **Vegetation**: -20 dB < σ⁰ < -5 dB  
- **Urban**: σ⁰ > -10 dB
- **Bare soil**: -25 dB < σ⁰ < -10 dB

**Reference**: Ulaby et al., "Microwave Radar and Radiometric Remote Sensing"

### 8.2 Incidence Angle Validation
Sentinel-1 IW mode valid ranges:
- **Minimum**: 29.1°
- **Maximum**: 46.0°

**Reference**: ESA Sentinel-1 Mission Requirements Document
